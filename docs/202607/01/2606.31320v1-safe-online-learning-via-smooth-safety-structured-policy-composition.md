## 摘要
安全在线强化学习要求策略在保持优化动态平滑的同时遵守安全约束。现有方法通常依赖于通过动作干预实现严格的安全执行，这会在系统交互和学习中引入不连续性；或者采用软安全约束公式，虽然保留了学习平滑性但提供的安全保证有限。我们提出了AutoSafe，一种将结构化安全监控和干预直接集成到动作生成过程中的安全感知策略架构。这种设计实现了性能驱动和安全保持行为之间的平滑、风险依赖的转换，从而产生连续的在线交互和学习动态。在一系列连续控制基准测试中的实证结果表明，AutoSafe在不牺牲学习平滑性的情况下实现了强大的安全执行。我们进一步在物理车杆系统上验证了AutoSafe，凸显了其在现实世界中安全在线学习的实践有效性。

## Abstract
Safe online reinforcement learning requires policies to respect safety constraints while maintaining smooth optimization dynamics. Existing approaches typically rely on either strict safety enforcement via action interventions, which introduce discontinuities in system interaction and learning, or soft safety constraint formulations, which preserve smooth learning but provide limited safety assurance. We propose AutoSafe, a safety-aware policy architecture that integrates structured safety monitoring and intervention directly into the action generation process. This design enables smooth, risk-dependent transitions between performance-driven and safety-preserving behaviors, resulting in continuous online interaction and learning dynamics. Empirical results across a suite of continuous-control benchmarks demonstrate strong safety enforcement without sacrificing learning smoothness. We further validate AutoSafe on a physical cart-pole system, highlighting its practical effectiveness for safe online learning in the real world.

## 1. 核心问题与整体含义（研究动机和背景）

安全在线强化学习面临一个根本性权衡：如何在保持策略优化动态平滑的同时，严格满足安全约束。现有方法主要分为两类：
- **硬干预方法**（如安全过滤器）：通过动作替换或投影直接强制执行安全，但会引入不连续性，阻断梯度传播，导致学习不稳定甚至发散。
- **软约束方法**（如约束策略优化、奖励塑造）：允许暂时违规以换取长期性能，但安全保证较弱，尤其不适合安全关键应用。

本文提出 **AutoSafe**，旨在通过一种可微的结构化策略组合，将安全监控和干预直接嵌入动作生成过程，从而实现性能驱动行为与安全保留行为之间的**平滑、风险依赖转换**，同时保证学习的连续性和稳定性。

## 2. 论文提出的方法论

### 核心思想
将学习策略 π\_θ 与一个认证安全的先验策略 π\_safe 通过**可微凸组合**融合，形成一个复合策略，其中组合权重 λ(s) 由状态相关的安全裕度动态决定。

### 关键技术细节
- **安全监控**：定义归一化安全裕度 Δ̃(s) ∈ [0,1]，代表当前状态相对于安全边界的距离。
- **可学习的干预权重**：使用指数型函数计算 λ(s)：
  ```
  λ(Δ̃, p) = (exp(p*(1 - Δ̃)) - 1) / (exp(p) - 1)
  ```
  满足 λ(1)=0（安全内部无干预）、λ(0)=1（边界时完全采用安全动作），且单调递减。锐度参数 p(s) 通过一个小型预测头学习，控制干预的缓急程度。
- **动作生成**：
  ```
  ̃a = (1 - λ(s)) * a_θ + λ(s) * a_safe
  ```
  其中 a_θ ∼ π\_θ(·|s)，a_safe = π\_safe(s)。此操作完全可微。
- **方差自动抑制**：当执行高斯策略时，复合策略的协方差变为：
  ```
  Σ̃(s) = (1 - λ(s))² * Σ_θ(s)
  ```
  即靠近边界时探索方差自动减小，防止偶然违规。
- **风险校准干预的可解形式**：通过局部仿射近似安全裕度，推导出满足机会约束的最小干预权重 λ* 的闭式表达式（式9），但实际实现中直接使用上述可学习的指数函数。
- **安全先验设计**：采用 Simplex 架构，通过线性矩阵不等式（LMI）求解 Lyapunov 矩阵 P 和反馈增益 F，获得一个认证的正不变安全椭球区域，提供确定性的安全动作 a_safe = Fs。

### 算法流程
以 Soft Actor-Critic (SAC) 为骨干，仅修改策略输出：
1. 从 π\_θ 采样 a_θ，由安全监控计算 Δ̃(s)。
2. 通过可学习 p(s) 生成 λ(s)。
3. 计算复合动作 ̃a，与环境交互。
4. 使用 ̃a 更新 Q 函数和策略参数（梯度可完全回传）。

## 7. 优点

- **结构简洁有效**：仅修改策略输出层，不改变 SAC 其余部分，易于集成。
- **可微性与理论属性**：凸组合保证可微性，同时量化了分布（均值偏移、方差抑制），为稳定学习提供理论基础。
- **不需要在线优化**：相比 CBF 或 MPC 类方法，计算开销低（与 SAC 相当），适用于高实时性系统。
- **自适应与鲁棒性**：通过可学习 p 适应不同任务；即使安全先验次优，也能通过调整 λ 限制其影响。
- **广泛验证**：覆盖模拟与实物，包含多种维度、动态环境及障碍规避场景。

## 8. 不足与局限

- **对模型知识的依赖**：安全先验（π\_safe、Lyapunov 矩阵 P）需要已知系统模型（线性化）并求解 LMI，对于复杂非线性系统设计困难。
- **局部线性化假设**：风险校准干预的推导依赖局部仿射近似，当安全裕度非线性强时可能不准确。
- **状态空间局限**：主要处理相对局部和结构化的状态约束；对于时变约束、外部干扰或移动障碍，需在线调整安全集（已展示初步扩展，但非通用方案）。
- **可扩展性未充分验证**：虽展示了高维任务（四足机器人），但更大规模和更复杂系统（如多机器人、人机交互）尚未实验。
- **基线对比的潜在偏差**：AutoSafe 与 SimplexRL 共享同一安全先验，而其他基线（如 CBF）的安全设计不同，可能不等价比较。此外，CBF 需要精细调参，否则经常不可行（论文已提及）。
- **真实世界实验规模小**：仅一个简单车杆系统，缺乏对真实噪声、时延、执行器故障等鲁棒性的评估。
