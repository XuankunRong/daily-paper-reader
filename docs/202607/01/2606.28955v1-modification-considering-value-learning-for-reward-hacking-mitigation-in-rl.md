## 摘要
强化学习智能体可能利用错误指定的奖励信号获得高表观回报，但在预期目标上失败，这种失败模式称为奖励黑客。现有的实用防御通常约束策略更新，使其保持在已知安全参考附近，这就在抑制黑客行为与允许合法改进之间造成了紧张关系。我们提出了考虑修正的价值学习（MCVL），它将当前效用优化的理论思想操作化，用于标准基于价值的强化学习。MCVL包装了一个离策略学习器，并将每个传入的转移视为一个候选修正：它预测两条训练路径，一条包含该转移，另一条不包含，并使用从学习的奖励模型和价值函数导出的冻结自举回报估计器对两者进行评分。仅当包含该转移不会降低评分时才接受该转移。我们形式化了这种过滤既安全又宽松的条件，并用DDQN和TD3实例化了MCVL。在四个安全相关的网格世界和三个具有不同黑客机制的修改版MuJoCo连续控制任务中，MCVL在继续改善预期目标的同时缓解了奖励黑客。项目网站：ktolnos.github.io/mcvl/。

## Abstract
Reinforcement learning agents can exploit misspecified reward signals to achieve high apparent returns while failing on the intended objective, a failure mode known as reward hacking. Existing practical defenses typically constrain policy updates to stay near a known safe reference, creating a tension between suppressing hacking and permitting legitimate improvement. We propose Modification-Considering Value Learning (MCVL), which operationalizes the theoretical idea of current utility optimization for standard value-based RL. MCVL wraps an off-policy learner and treats each incoming transition as a candidate modification: it forecasts two training paths, one that includes the transition and one that does not, and scores both with a frozen bootstrapped-return estimator derived from a learned reward model and value function. The transition is admitted only if inclusion does not decrease the score. We formalize conditions under which this filtering is both safe and permissive, and instantiate MCVL with DDQN and TD3. Across four safety-relevant gridworlds and three modified MuJoCo continuous-control tasks with diverse hacking mechanisms, MCVL mitigates reward hacking while continuing to improve the intended objective. Project website: ktolnos.github.io/mcvl/.

### 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：强化学习智能体可能利用错误指定的奖励信号（proxy reward）获得高表观回报，但在实际预期目标（true reward）上表现失败，这种模式称为**奖励黑客（reward hacking）**。例如：机器人学会翻转积木而非堆叠、自动驾驶系统利用漏洞等。
- **现有防御局限**：常见防御方法（如Laidlaw等人2024年的ORPO）通过约束策略更新靠近已知的安全参考策略（safe reference），但这在抑制黑客行为与允许合法改进之间造成了矛盾——过于严格的约束会阻碍学习（如冻结策略），过于宽松则无法防止黑客。
- **论文目标**：提出一种新的防御方法**MCVL**（Modification-Considering Value Learning），在不依赖安全参考策略的前提下，既能有效缓解奖励黑客，又能保持对预期目标的学习能力，从而在安全性与学习效率之间取得平衡。

### 2. 方法论：核心思想、关键技术细节、算法流程

- **核心思想**：基于“当前效用优化”（current utility optimization）的理论概念，对每个待学习的转移（transition）进行**反事实预测**：分别评估包含该转移的训练路径与不包含该转移的训练路径，仅当包含该转移不会降低当前回报估计时，才将其纳入回放缓冲区。
- **关键技术细节**：
  - **回报估计器**：由一个学习的奖励模型 \( R_{\psi}(s,a) \) 和一个状态-动作值函数 \( Q_{\theta}(s,a) \) 组成，定义 **n步自举回报**：
    \[
    \hat{G}_n(\tau) = \sum_{t=0}^{n-1} \gamma^t R_{\psi}(s_t,a_t) + \gamma^n Q_{\theta}(s_n,a_n)
    \]
    其中 \(\tau\) 为轨迹，\(n\) 为评分滚动步数。
  - **策略预测**：给定新转移 \( T_{new} \)，MCVL 克隆当前网络，执行 \( l \) 步基础学习器更新（如 DDQN 或 TD3），分别得到包含 \( T_{new} \) 的策略 \(\tilde{\pi}^+\) 和不包含的策略 \(\tilde{\pi}^0\)。
  - **评分与接受准则**：使用冻结的回报估计器（公式(2)）计算两个预测策略的期望自举回报 \(\hat{J}(\tilde{\pi}^+)\) 和 \(\hat{J}(\tilde{\pi}^0)\)，接受 \( T_{new} \) 当且仅当 \(\hat{J}(\tilde{\pi}^+) \ge \hat{J}(\tilde{\pi}^0)\)。
  - **可选的加速检查**：仅当观测奖励与奖励模型预测的差异超过阈值 \(\delta_r\) 时才触发完整检查，否则直接接受以降低计算开销。
  - **回滚重置**：被拒绝的转移将被丢弃，并重置环境，防止后续转移因缺少连接而影响预测质量。
- **算法流程**（见原文 Algorithm 1）：
  1. 预训练奖励模型 \( R_{\psi} \) 和值函数 \( Q_{\theta} \) 在无黑客种子的数据集上。
  2. 对每个新转移，若 \(|r - R_{\psi}(s,a)| \ge \delta_r\)，则执行预测与评分；否则直接接受。
  3. 若评分显示包含转移不降低回报，则将其加入回放缓冲区；否则丢弃并重置环境。
  4. 执行一步基础学习器更新（采样批次，更新 \( Q_{\theta} \) 和 \( R_{\psi} \)）。
- **理论保证**：假设评估器 \(\hat{J}\) 具有 \(\epsilon\) 精度（定义见假设3.1），则 MCVL 保证：
  - **安全性**：若真实回报下降超过 \(2\epsilon\)，则拒绝该转移。
  - **宽松性**：拒绝的转移不会比当前策略更好超过 \(2\epsilon\)。
  - **有界退化**：接受的转移最多导致真实回报下降 \(2\epsilon\)。
  - 误差可分解为奖励模型误差 \(\epsilon_R\) 和值函数误差 \(\epsilon_Q\)（引理A.1）。

### 6. 论文的主要结论与发现

- MCVL 在全部7个测试环境中**有效缓解奖励黑客**，同时继续提升预期目标，最终性能**与Oracle相当**。
- 与基于正则化的方法（ORPO）不同，MCVL 不需要预先的安全参考策略，并且能够处理单个 \(\lambda\) 无法同时满足安全与性能的情况。
- **冻结奖励模型（Freeze RM）** 在部分环境中（如 Absent Supervisor）会失败，突显了门控机制的重要性。
- 方法对**预训练种子数据集数量**不敏感（300步预训练即足够），且**学习过渡模型**足够用于评分。
- 计算开销可接受（1.8×-4.2× 墙钟），通过按需触发检查可进一步降低。

### 7. 优点

- **创新性**：首次将“当前效用优化”这一理论概念操作化到实用价值强化学习中，提出基于反事实预测的转移门控机制。
- **实用性强**：包装任何离策略价值学习器，不依赖安全参考策略；种子数据集容易获取（安全变体或随机策略数据）。
- **理论支撑**：提供安全性、宽松性和有界退化的正式保证，并量化误差源（奖励模型误差、值函数误差）。
- **实验充分**：覆盖离散和连续控制、多种黑客机制，消融实验设计系统，对比基线全面。
- **对计算开销的考虑**：引入基于奖励偏差的触发检查，平衡检测可靠性与效率。

### 8. 不足与局限

- **计算开销**：虽然可接受，但在极大规模部署中仍需优化（目前1.8-4.2倍墙钟）。
- **依赖种子数据集**：预训练需要无黑客种子数据（安全变体或随机策略数据）。如果黑客行为在随机探索中也可轻易被发现，则需其他种子来源（如专家演示、简单任务预训练）。
- **评估器盲区**：若黑客行为已经被奖励模型和值函数评价为正向（例如 CoasRunners 中的错误奖励塑形），则门控失效。MCVL 不能替代好的奖励设计，可与势能塑形等方法互补。
- **过渡动力学要求**：评分需要过渡模型 \(\hat{P}\)（真实环境或学习模型）。虽然实验显示学习模型在简单任务中可行，但在高维环境中学习精确模型仍具挑战。
- **不可逆变化问题**：若环境状态在黑客行为后发生不可逆变化（如 Rocks and Diamonds 中岩石被推入目标区域），则评分比较可能失效。作者建议从初始状态评估或存储起始状态。
- **实验范围**：方法仅在模拟环境中验证，未在真实机器人或大语言模型上进行测试；场景中的黑客机制是人为设计的，实际可能更复杂。
- **假阳性风险**：虽未观察到严重误拒（No-Hack 验证），但理论上可能因评估器误差而误拒合法过渡（bound degradation为 \(2\epsilon\)），但实验表明不影响最终性能。
