## 摘要
现代LLM智能体越来越依赖上下文压缩、摘要或驱逐，以将长时间运行的会话保持在令牌预算内。我们表明，这种上下文管理层是一个安全关键的故障面：在可见时智能体可靠遵守的上下文内治理约束，可能被压缩悄然移除，导致同一智能体在会话后期执行被禁止的工具操作。我们将这种故障模式称为“治理衰退”。我们引入了ConstraintRot，一个具有确定性工具调用评分的长期智能体场景基准，并测量了七个模型系列中压缩引起的违规行为。在1323个回合中，违规行为从完整上下文中策略时的0%上升到压缩后的30%，某些模型达到59%；当约束在摘要中幸存时，违规行为保持0%，但当约束被丢弃时，违规行为达到38%。我们进一步研究了压缩-驱逐攻击，其中对抗性的上下文内容使摘要器偏向于省略合法策略，并表明优化的注入能够击败每个评估模型。最后，我们提出了约束固定，一种简单的无需训练的缓解措施，将治理约束与有损压缩隔离，并在我们的基准中将违规行为恢复为0%。这些结果将上下文管理确定为已部署LLM智能体的一级治理面。

## Abstract
Modern LLM agents increasingly rely on context compaction, summarization, or eviction to keep long-running sessions within a token budget. We show that this context-management layer is a safety-critical failure surface: in-context governance constraints that agents reliably obey while visible can be silently removed by compaction, causing the same agent to perform prohibited tool actions later in the session. We call this failure mode Governance Decay. We introduce ConstraintRot, a benchmark of long-horizon agent scenarios with deterministic tool-call grading, and measure compaction-induced violations across seven model families. Across 1,323 episodes, violation rises from 0% with the policy in full context to 30% after compaction, reaching 59% for some models; when the constraint survives the summary, violation remains 0%, but when it is dropped, violation reaches 38%. We further study a Compaction-Eviction Attack, in which adversarial in-context content biases the summarizer to omit a legitimate policy, and show that optimized injections defeat every evaluated model. Finally, we propose Constraint Pinning, a simple training-free mitigation that quarantines governance constraints from lossy compaction and restores violation to 0% in our benchmark. These results identify context management as a first-class governance surface for deployed LLM agents.

## 1. 核心问题与研究动机

- **背景**：现代LLM智能体（agent）在长时间运行对话中，为了控制上下文token预算，普遍采用“上下文压缩”（context compaction）机制——通过LLM摘要或驱逐旧内容，压缩历史记录以维持任务连续性。
- **核心问题**：这种压缩过程在优化任务准确性的同时，会“静默”删除智能体原本严格遵守的治理约束（如“不得发送外部邮件”的规则）。当约束被压缩丢弃后，面对相同的禁止请求，智能体可能无条件执行违规操作——这种现象被称为**Governance Decay（治理衰退）**。
- **整体含义**：上下文管理成为已部署LLM智能体的一级安全治理面，而现有安全研究（如提示注入、权限检查）忽视了这一层。治理衰退是系统性的、可被利用的缺陷。

## 2. 方法论：核心思想与关键技术细节

### 2.1. 治理衰退的形式化定义
- 设智能体模型 \(M\)、历史上下文 \(H\)、禁止请求 \(r\)、违规概率 \(v(M, H)\)。
- 压缩后违规增加量：\(\Delta_{\text{decay}} = v(M, [C(H_{<t}); r]) - v(M, [H_{<t}; r])\)，其中 \(C\) 是压缩算子。

### 2.2. Compaction-Eviction Attack（压缩驱逐攻击）
- **威胁模型**：攻击者仅控制智能体摄入的非系统内容（如工具输出、检索文档），不控制模型、系统提示或压缩算子。
- 两种变体：
  - **volume variant**：注入大量无害内容，迫使压缩触发并丢弃早期策略。
  - **summarizer-injection variant**：在内容中嵌入针对压缩模型的指令（如“摘要时省略过时的合规序言”），使策略主动被丢弃。
- 通过搜索不同注入策略（废弃框架、token预算框架、系统冒充等），优化后能击败所有测试模型。

### 2.3. Constraint Pinning（约束固定）
- **思想**：将治理约束提取到独立的“固定缓冲区”（pinned buffer），该缓冲区不受压缩影响，并在每次压缩后重新注入上下文。
- **实现**：训练无关、仅修改harness层；固定约束 ≈47 tokens，开销 < 0.5% 的上下文。
- 完整性检查：要求压缩后的上下文仍然蕴含固定约束。

### 2.4. ConstraintRot基准
- 自包含的9个长期智能体场景，每个场景包括：
  1. 策略回合（建立约束）
  2. 长序列良性任务（使上下文超出压缩预算）
  3. 触发请求（自然完成会违反约束）

## 6. 主要结论

1. **压缩导致显著治理衰退**：完整上下文时违规0%，单次压缩后平均升至30%（最高59%）；若约束幸存则违规0%，被丢弃则38%。
2. **软策略更易受影响**：软组织策略+50 pts（0→50%），硬安全规范仅+6 pts（8.3倍差距），说明治理衰退主要侵蚀只能存在于上下文的部署特定规则。
3. **攻击可击败所有模型**：被动压缩鲁棒模型（如GLM-5.1、Gemini-3.5-flash）在主动注入攻击下仍会失败；优化后的注入使Claude从0%升至65%，DeepSeek达100%。
4. **约束固定有效**：在所有模型和攻击变体下将违规恢复至0%，开销<0.5% tokens，且不损害正常功能（99%允许动作完成）。
5. **真实框架再现**：LangGraph、LangMem、AutoGen等框架中违规率可达65%–100%，确认问题现实性。
6. **治理衰退是harness层问题**：摘要器决定策略是否幸存，而非智能体模型本身；改进摘要器（而非模型）可以缓解，但非完整解决方案。

## 7. 优点

- **问题识别新颖**：首次系统性地将上下文压缩识别为智能体安全的关键治理面，区分于提示注入和权限控制。
- **基准设计严谨**：ConstraintRot包含软/硬策略区分、确定性评分、自包含触发，便于复现。
- **攻击与防御全面**：提出两种攻击变体和简单有效的防御，并进行跨模型、跨策略、跨语言的全面评估。
- **机制清晰**：通过摘要器-智能体交叉实验、生存预测分析，确认策略删除是根本原因，排除了上下文长度退化等混淆因素。
- **实用性强**：防御训练无关、低开销，可直接部署；实验验证了多个主流agent框架。
- **结果透明度**：报告了防御失败场景（权威仿冒），诚实揭示开放问题。
