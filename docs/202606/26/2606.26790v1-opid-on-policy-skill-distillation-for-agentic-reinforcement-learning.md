## 摘要
基于结果的强化学习为语言智能体提供了稳定的优化基础，但其稀疏的轨迹级奖励几乎无法指导哪些中间决策应被加强或抑制。在策略自蒸馏提供了密集的词元级监督，然而现有技能条件化变体往往依赖于外部技能记忆或检索到的特权上下文，这些方法维护成本高昂，且在多轮交互中容易与当前策略导致的状态分布不匹配。我们提出OPID（在策略技能蒸馏），一个直接从已完成在策略轨迹中提取技能监督的框架。OPID将轨迹事后归因表示为层次化技能：情节级技能捕捉全局工作流或失败规避规则，而步骤级技能捕捉关键时间步的局部决策知识。一个关键优先路由机制在识别出关键决策时使用步骤级技能，否则回退到情节级技能作为默认指导。所选技能被注入交互历史，使旧策略能够在原始上下文和技能增强上下文下对相同的采样响应重新评分。由此产生的对数概率偏移形成了词元级自蒸馏优势，并与结果优势相结合进行策略优化。OPID因此保留了RL作为主要训练目标，同时引入了密集且分布匹配的事后监督。在ALFWorld、WebShop和基于搜索的QA上的实验表明，OPID在智能体性能、样本效率和鲁棒性上普遍优于纯结果RL和现有的技能蒸馏基线。我们的代码可在https://github.com/jinyangwu/OPID/tree/main获取。

## Abstract
Outcome-based reinforcement learning provides a stable optimization backbone for language agents, but its sparse trajectory-level rewards provide little guidance on which intermediate decisions should be reinforced or suppressed. On-policy self-distillation offers dense token-level supervision, yet existing skill-conditioned variants often rely on external skill memories or retrieved privileged context, which are costly to maintain and can be mismatched with the state distribution induced by the current policy in multi-turn interaction. We propose \textbf{OPID} (\textbf{O}n-\textbf{P}olicy Sk\textbf{i}ll \textbf{D}istillation), a framework that extracts skill supervision directly from completed on-policy trajectories. OPID represents trajectory hindsight as hierarchical skills: episode-level skills capture global workflows or failure-avoidance rules, while step-level skills capture local decision knowledge at critical timesteps. A critical-first routing mechanism uses step-level skills when critical decisions are identified and falls back to episode-level skills as default guidance otherwise. The selected skill is injected into the interaction history, allowing the old policy to re-score the same sampled response under both original and skill-augmented contexts. The resulting log-probability shift yields a token-level self-distillation advantage, which is combined with the outcome advantage for policy optimization. OPID thus preserves RL as the primary training objective while introducing dense, distribution-matched hindsight supervision. Experiments on ALFWorld, WebShop and Search-based QA demonstrate that OPID generally improves agent performance, sample efficiency, and robustness over outcome-only RL and existing skill-distillation baselines. Our code is available at https://github.com/jinyangwu/OPID/tree/main.

### 2. 方法论：核心思想、关键技术细节、算法流程（文字说明）

- **核心思想**：将每一段完整的在线轨迹转化为两种层次的自然语言技能——**情节级技能（episode-level skill）** 概括整个轨迹的工作流程或失败规避规则；**步骤级技能（step-level skill）** 捕捉关键时间步的局部决策知识。然后通过**关键优先路由（critical-first routing）** 机制，对每个决策步骤选择最合适的技能（关键步骤采用步骤级技能，其余采用情节级技能），将所选技能注入交互历史，让旧策略在原始上下文和技能增强上下文下对同一采样响应重新评分，得到 token 级的对数概率偏移，即**技能自蒸馏优势** \(A^{\text{skill}}\)。最终优势为 \(A^{\text{OPID}} = A^{\text{ep}} + \lambda_{\text{skill}} A^{\text{skill}}\)，用于标准裁剪策略目标（类似 GRPO/PPO）优化。

- **关键技术细节**：
  1. **情节级技能提取**：通过 LLM 分析器 \(A\) 将完整轨迹（包括任务提示、步骤观测、动作、结果）映射为一段自然语言描述的工作流或失败规避规则。
  2. **步骤级技能提取**：分析器识别轨迹中的关键时间步集合 \(C_\tau\)，并为每个关键步生成一句面向行动的指导技能。
  3. **关键优先路由**：\(s_{\tau,t} = \begin{cases} s^{\text{step}}_{\tau,t}, & t \in C_\tau \\ s^{\text{ep}}_\tau, & \text{otherwise} \end{cases}\)，避免盲目组合两种技能。
  4. **技能注入与自蒸馏优势计算**：用确定性函数 \(H(h_{\tau,t}, s_{\tau,t})\) 将路由技能追加到历史前或后，得到增强上下文 \(\tilde{h}_{\tau,t}\)；旧策略 \(\pi_{\theta_{\text{old}}}\) 分别计算原始上下文和增强上下文下出现 token \(y_{\tau,t,\ell}\) 的对数概率，差值即为 \(A^{\text{skill}}_{\tau,t,\ell}\)。
  5. **策略优化**：采用带裁剪的重要性比率的策略梯度损失，结合结果优势（GRPO 风格的组归一化）和技能优势，加上 KL 正则项。

### 7. 优点

- **无需外部技能库或检索**：训练时只从当策略自己的轨迹中提取技能，推理时无额外开销，避免了分布偏移和维护成本。
- **与结果 RL 自然融合**：保留 GRPO 作为主目标，技能优势作为辅助信号，使训练既稳定又细致。
- **层次化设计+关键优先路由**：兼顾全局稳定性和局部精确性，通过路由避免两种信号冲突。
- **广泛的实验验证**：涵盖具身、网页、搜索三种长期交互场景，使用多个模型尺度，并与众多基线公平对比。
- **理论分析支撑**：附录提供反向 KL 分解、上下文分布匹配、路由最优性等理论结果，增强了方法的可解释性。

### 8. 不足与局限

- **实验覆盖有限**：仅测试了 ALFWorld、WebShop 和 Search-based QA，未涉及更复杂的智能体基准（如 SWE-bench、WebArena、VisualWebArena、OdysseyArena 等），泛化性有待验证。
- **对技能分析器（Analyzer）的依赖**：虽然推理时不依赖，但在训练中需要额外的 LLM（GLM-5.2）来分析轨迹并生成技能，增加了训练计算成本和延迟。论文未分析此分析器的开销或对其质量进行深入消融。
- **技能数量的控制**：步骤级技能数量在实现中被硬限制（如 ALFWorld/WebShop 最多 5 个，Search-based QA 最多 2 个），可能忽略某些需要更多关键步骤的任务。
