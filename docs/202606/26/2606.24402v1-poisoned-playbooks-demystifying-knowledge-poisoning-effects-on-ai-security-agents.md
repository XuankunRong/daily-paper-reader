## 摘要
AI安全代理日益依赖检索增强生成（RAG）来利用外部安全知识进行漏洞分析与利用推理。这带来了一种新风险：被投毒的文章可被转化为错误的利用行为。然而，先前关于RAG投毒的研究主要关注问答场景下的答案篡改，而对采取行动的安全代理知之甚少。本文旨在通过针对真实世界挑战和AI代理的精心构造的投毒数据来揭示此类特性。首先，我们展示了如何通过将精心构造的单一投毒文章注入被称为“有毒剧本”的公共风格安全知识源，来改变基于RAG的AI安全代理的行为。在11个CTF挑战、3个前沿LLM系列、2代模型以及11个真实世界CVE中，我们发现投毒采纳是系统性的而非随机性的。为解释这一模式，我们引入了验证边界（VB），这是一种基于代理可用于反驳检索到的声明之证据的三级经验分类。最后，我们评估了验证提示与多源检索，结果表明两者在存在更强证据时均有帮助，但在证据稀疏与零日条件下效果减弱。

## Abstract
AI security agents increasingly rely on Retrieval-Augmented Generation (RAG) to use external security knowledge for vulnerability analysis and exploit reasoning. This creates a new risk: poisoned write-ups can be operationalized into incorrect exploit behavior. Yet, prior work on RAG poisoning has mostly studied answer corruption in QA settings, much less is known about action-taking security agents. This paper aims to reveal such characteristics with crafted poisons about real-world challenges and AI agents. First, we demonstrate how a crafted single poisoned write-up injected into public-style security knowledge sources which we denote as Poisoned Playbooks, alters the behavior of RAG-based AI security agents. Across 11 CTF challenges, 3 frontier LLM families, 2 model generations, and 11 real-world CVEs, we find that poison adoption is systematic rather than random. To explain this pattern, we introduce the Verification Boundary (VB), a 3-level empirical classification based on what evidence the agent can use to refute a retrieved claim. Finally, we evaluate verification prompting and multi-source retrieval, showing that both help when stronger evidence exists, but weaken under sparse-evidence and zero-day conditions.

## 1. 核心问题与整体含义（研究动机和背景）

- **研究动机**：AI安全代理（如用于漏洞分析、渗透测试的LLM代理）越来越多地依赖检索增强生成（RAG）来从外部安全知识源（如漏洞利用仓库、GitHub、博客文章）获取实时情报。然而，这些公共知识源缺乏严格审查，容易被攻击者植入恶意内容。现有RAG投毒研究主要关注问答场景下的答案篡改，而忽略了行动型安全代理——代理不仅输出文本，还会基于检索到的知识执行具体操作（如发起攻击路径）。因此，需要系统性地研究知识投毒对安全代理行为的影响。
- **核心问题**：攻击者能否通过向公共安全知识源注入单条精心构造的“有毒剧本”（Poisoned Playbook）来系统性地改变RAG安全代理的利用行为？哪些因素决定代理是拒绝还是采纳有毒声明？现有防御措施的效果及其局限性如何？
- **整体意义**：揭示了安全代理供应链中一个被忽视的脆弱性——推理阶段的知识投毒，并提出了可解释的分类框架（验证边界VB），为构建更鲁棒的安全代理提供指导。

## 2. 方法论

- **核心思想**：设计一套研究框架，包括构造有毒文章（Poisoned Playbook）、建立RAG安全代理测试平台、定义毒性采纳率（PAR）指标，以及提出验证边界（VB）三级分类来解释不同场景下代理的行为差异。
- **关键技术细节**：
  - **Poisoned Playbook**：一种模仿真实安全写作文风的伪造文章，嵌入攻击者希望代理采纳的虚假声明（如声称某个漏洞被某种机制防御，从而引导代理放弃正确利用路径）。每个剧本30-60分钟即可人工生成。

  - **验证边界 (Verification Boundary, VB)**：根据代理能否通过可用证据反驳检索到的声明，将声明分为三级：
    - **L1 (代码可验证)**：可从本地源码/配置直接验证，代理几乎总是拒绝。
    - **L2 (知识可验证)**：本地证据不足但模型参数知识可判断，结果随模型能力变化。
    - **L3 (运行时依赖)**：需要外部运行时信息（如特定版本库行为），代理无法从现有证据或模型知识中获取，几乎总是采纳。

## 6. 主要结论与发现

- **单条有毒文章可系统性改变安全代理行为**：在oracle检索下，部分被拒、部分因模型能力差异而改变、部分被一致采纳。在E2E全管道中，有毒文章能在排名中位居前列并改变代理策略。
- **跨模型/跨代普遍性**：在信息垄断下，所有模型家族（Claude、GPT、Gemini）均采纳有毒声明；更强的模型（Opus 4.6 vs Opus 4）能拒绝部分之前采纳的声明（L2区域扩展），但L3声明仍被一致采纳。
- **验证边界（VB）有效解释采纳模式**：L1（代码可验证）接近0%采纳；L2（知识可验证）模型依赖；L3（运行时依赖）100%采纳。真实CVE验证了相同模式：已知漏洞（有强竞争证据）被拒绝，后截止运行时依赖漏洞（如React2Shell、SharePoint等）被一致采纳。
- **信息垄断是强预测因子**：当代理只有一个有毒来源且缺乏本地矛盾证据和模型知识时，采纳几乎必然；在稀疏证据和零日条件下威胁最大。
- **防御措施有效但有限**：验证提示和多源检索在有更强证据时能显著降低采纳率（如在5个运行时依赖CTF中从100%降至~0%），但在完全信息垄断或缺乏正确竞争证据时无效。防御无法创造新证据，只能重新加权现有证据。
- **实用性**：有毒文章生成成本低（30-60分钟），且具有一对多的可扩展性，一个变体能影响任意下游代理。

## 7. 优点

- **问题新颖且重要**：聚焦于行动型安全代理的知识投毒，填补了现有RAG投毒研究（主要针对QA答案篡改）的空白。
- **系统性实证研究**：跨越多个模型家族、代际、真实漏洞，结果具有广泛代表性。

- **防御评估区分有效条件**：明确指出验证提示和多源检索在强证据下有效但在弱证据/零日下失效，避免了过度乐观。
