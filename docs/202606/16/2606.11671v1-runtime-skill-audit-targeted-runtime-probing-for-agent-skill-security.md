## 摘要
智能体技能让大语言模型智能体能够重用指令、资源、工具和工作流程，但也为恶意行为创造了新的藏身之处。一项技能可能在文档或代码中看似无害，但只有在与特定用户请求、本地资产、持久状态或多步工具交互一起调用时才会变得有害。这使得纯静态审查变得脆弱。我们提出了运行时技能审计（RSA），一种动态分析方法，通过询问技能中介的智能体在目标运行时条件下实际做了什么来审计技能。RSA不是用相同的通用任务测试每个技能，而是分析风险相关接口，准备执行这些接口所需的上下文，并根据产生的追踪证据分配安全标签。我们在OpenClaw上实现了RSA，并在100个技能上对其进行了评估，与代表性静态基线进行了对比。RSA达到了90.0%的准确率，88.0%的真阳性率和8.0%的假阳性率，比最佳静态基线提高了13.0个百分点。在自进化攻击下，静态检测器在一两轮后崩溃，而RSA在每一轮中继续检测到20个恶意技能中的19-20个。

## Abstract
Agent skills let LLM agents reuse instructions, resources, tools, and workflows, but they also create a new place for malicious behavior to hide. A skill may look benign in its documentation or code while becoming harmful only when it is invoked with particular user requests, local assets, persistent state, or multi-step tool interactions. This makes purely static vetting brittle. We present Runtime Skill Audit (RSA), a dynamic analysis method that audits skills by asking what the skill-mediated agent actually does under targeted runtime conditions. Instead of testing every skill with the same generic tasks, RSA profiles risk-relevant interfaces, prepares the execution context needed to exercise them, and assigns security labels from the resulting trace evidence. We instantiate RSA on OpenClaw and evaluate it on 100 skills against representative static baselines. RSA achieves 90.0\% accuracy with an 88.0\% true positive rate and an 8.0\% false positive rate, improving accuracy by 13.0 percentage points over the best static baseline. Under self-evolving attacks, static detectors collapse after one or two rounds, while RSA continues to detect 19--20 out of 20 malicious skills across rounds.

## 2. 方法论：核心思想与关键技术细节

### 核心思想
RSA（Runtime Skill Audit）是一种动态分析方法，通过三个关键步骤实现行为导向的安全审计：
1. **风险引导的任务生成**：基于技能工件推断其可能使用的工具接口，并映射到安全风险类别，生成针对性的运行时探测任务。
2. **上下文感知的运行时执行**：在沙盒环境中准备与探测任务匹配的上下文（如文件、凭证、持久状态），然后执行技能，收集结构化操作追踪。
3. **追踪证据驱动的安全标签**：基于实际执行轨迹（工具调用、文件访问、网络请求等）而非工件表面特征，输出安全标签（benign executed / harmful blocked / harmful executed / uncertain）。

### 关键技术细节
- **技能分析**：使用LLM分析技能工件，推断其可能调用的平台工具（如文件读写、shell执行、网络请求、内存操作等），并映射到预定义的工具分类法。
- **任务生成**：根据分析结果生成包含用户请求、沙盒准备文件、执行约束的结构化任务，确保探测器能覆盖风险接口。
- **沙盒执行与修复**：在隔离环境中执行任务；若执行因环境问题失败，通过有限步数的修复循环（如更新任务、修改权限、切换镜像）尝试恢复，避免将无效执行误判为良性。
- **知识库与记忆**：
  - **知识库**：固定资源，包含工具分类法、风险映射、攻击模式模板、标签定义、评估清单，灵感来自MITRE ATT&CK。
  - **记忆**：从先前执行中总结有效触发条件、误报模式、漏检行为，存储为紧凑摘要，用于指导后续分析。
- **标签规则**：即使智能体阻止了恶意负载，只要追踪暴露了恶意分支，技能仍被标记为“harmful blocked”，区分“存在恶意行为”与“实际执行”。

## 6. 主要结论与发现

- **RSA在100技能数据集上**：
  - 准确率90.0%，TPR 88.0%，FPR 8.0%，比最佳静态基线SkillScanner的77.0%准确率提升13个百分点，同时FPR从12.0%降至8.0%。
- **自进化攻击下**：
  - 静态检测器（Aguara、SkillScanner）在1-2轮后完全失效（检测率降为0）。
  - RSA稳定保持19-20/20的检测率，因为恶意行为必须在执行中暴露，无法通过文本改写隐藏。
- **知识引导的有效性**：添加知识将TPR从82.0%提升至88.0%，FPR从16.0%降至8.0%，表明知识帮助生成更精准任务并更一致地解释追踪证据。
- **案例分析证明**：RSA能从实际执行追踪中识别出数据窃取分支，即使智能体阻止了执行，仍能标记技能为恶意（harmful blocked）。

## 7. 优点

- **动态行为审计**：直接基于运行时证据（工具调用、文件访问、网络请求）判断，克服静态分析对上下文依赖攻击的无力。
- **风险引导的探测**：针对每个技能的风险接口生成任务，避免通用测试的盲目性，提高检测效率。
- **上下文感知执行**：通过准备特定本地资产、凭证、状态，暴露仅在实际环境中才出现的恶意分支。
- **自修复与虚拟shim**：减少因环境不完整导致的无效执行，提升可操作性。
- **知识+记忆增强**：利用外部安全知识和历史执行经验指导分析，同时最终判断仍基于当前追踪，保持可解释性。
- **公平对比**：与多种代表性静态方法进行系统比较，并评估自适应攻击下的鲁棒性。

## 8. 不足与局限

- **数据集规模有限**：仅100技能，且恶意技能改编自现有基准，未涵盖所有可能攻击模式（如多步长时间跨度的攻击）。
- **平台通用性不足**：仅在OpenClaw上验证，成果向其他智能体平台（如Claude Code、Codex）迁移时需重新适配工具分类法和执行接口。
- **依赖LLM质量**：分析、任务生成、判断均依赖LLM，其准确性受模型能力、提示设计影响，存在一致性风险。
- **非穷举性**：生成的探测任务可能遗漏某些触发条件或外部状态，技能仍可能在其他场景下暴露恶意行为。RSA应视为行为测试方法而非完整安全证明。
- **计算资源未报告**：缺乏对RSA运行效率的分析，可能难以在实时或大规模生态中部署。
- **消融实验深度不足**：未单独分析知识库与记忆各自的贡献，也未改变LLM模型进行稳健性测试。
