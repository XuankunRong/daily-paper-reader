## 摘要
智能体强化学习已成为将大语言模型从静态聊天机器人转变为交互式智能体的重要后训练范式，催生了诸如 OpenClaw 等代表性应用。现有工作主要关注策略优化算法和训练框架，但对智能体-环境交互的完整数据生命周期（从数据生成到训练消费）关注不足。为弥补这一空白，我们提出了 Claw-R1，一个面向智能体强化学习的交互式步级数据中间件系统。Claw-R1 通过两个核心组件（网关服务器和数据池）连接异构智能体运行环境与强化学习训练后端。网关服务器通过统一的大语言模型 API 入口捕获多轮交互步骤，而数据池将这些步骤组织成由提示 ID、响应 ID、奖励及其他元数据构成的步级记录。在我们的演示中，用户可以交互式检查实时轨迹，查看每一步的状态、动作和奖励，根据质量和就绪状态整理数据，并为不同的下游强化学习算法配置训练就绪批次。总体而言，Claw-R1 将智能体交互痕迹视为受管理的数据资产，而非临时运行时日志。通过本次演示，我们希望鼓励社区认识到数据管理在智能体强化学习中的重要性。我们的代码可在 https://github.com/AgentR1/Claw-R1 获取，演示视频见链接 https://youtu.be/Pw47dAOw6B0。

## Abstract
Agentic reinforcement learning (RL) has become an important post-training paradigm for turning LLMs from static chatbots into interactive agents, giving rise to representative applications such as OpenClaw. Existing work mainly focuses on policy optimization algorithms and training frameworks, but pays less attention to the full data lifecycle of agent-environment interactions, from data production to training consumption. To bridge this gap, we present Claw-R1, an interactive step-level data middleware system for agentic RL. Claw-R1 connects heterogeneous agent runtimes with RL training backends through two core components: a Gateway Server and a Data Pool. The Gateway Server captures multi-turn interaction steps through a unified LLM API entry point, while the Data Pool organizes them into step-level records consisting of prompt IDs, response IDs, rewards and other metadata. In our demo, users can interactively inspect live trajectories, examine the state, action, and reward of each step, curate data by quality and readiness, and configure training-ready batches for different downstream RL algorithms. Overall, Claw-R1 treats agent interaction traces as managed data assets rather than temporary runtime logs. Through this demonstration, we hope to encourage the community to recognize the importance of data management in agentic RL. Our code is available at https://github.com/AgentR1/Claw-R1 and the demonstration video can be found at link https://youtu.be/Pw47dAOw6B0.

## 2. 论文提出的方法论：核心思想、关键技术细节、算法流程

- **核心思想**：以数据为中心，将 Agentic RL 视为**将运行时交互数据转化为训练可消费资产**的过程。通过两种核心组件实现解耦：
  - **Gateway Server（网关服务器）**：为智能体运行时提供与 OpenAI 兼容的 LLM API 入口，统一捕获多轮交互步骤（对黑盒智能体捕获 LLM 调用，对白盒智能体接受显式步骤提交，对黑盒服务接收 HTTP 事件和人工反馈）。
  - **Data Pool（数据池）**：存储**步级记录**，每条记录包含 prompt ID、response ID、奖励、元数据（轨迹关系、策略版本、来源等），并构建索引，支持按 ID、轨迹、奖励状态、就绪状态等快速检索。
- **关键技术细节**：
  - **步级数据抽象**：将智能体执行建模为步级 MDP：每一步 \((s_t, a_t, r_t, s_{t+1})\)，记录标准化格式，去除运行时特定细节，仅保留输入、输出、奖励信息，实现运行时与训练后端完全解耦。
  - **设计原则**：
    1. 低侵入式采集：无需统一执行框架或侵入式修改现有智能体。
    2. 步级原生表示：保留交互结构和 RL 语义，同时可访问底层 token 序列用于回放和优化。
    3. 异步解耦：数据收集与模型训练分离，支持持续运行的异构智能体工作负载。
    4. 后端感知服务：中间件独立于具体 trainer，通过轻量适配层和标准化数据抽象暴露训练器兼容接口。
  - **数据优化**：利用前缀树合并（Prefix-tree Merging）——多个轨迹共享长上下文时，将共享 token 前缀合并为一个节点，不同分支保留为独立子节点，减少冗余长上下文计算，同时保持每个原始步骤的奖励、轨迹关系和训练语义。
- **算法流程（文字说明）**：
  1. 智能体运行时通过 Gateway Server 提交/捕获交互事件。
  2. Gateway Server 将事件标准化为步级记录（含 prompt ID, response ID, reward, metadata），推送至 Data Pool。
  3. Data Pool 存储记录，构建索引，并可通过前缀树合并进行数据优化。
  4. RL 训练后端通过拉取式（pull-based）批量接口从 Data Pool 获取训练就绪批次，可基于奖励可用性、策略新鲜度、轨迹完整性、质量标签等过滤数据。
  5. 训练后端消费数据执行策略优化（如 PPO, GRPO 等），训练过程中 Gateway Server 继续持续采集新数据。
