## 摘要
在高风险领域，智能体安全性需要正式的策略执行，但现有大多数方法要么依赖于概率性护栏（微调分类器、基于提示的引导）——这些方法无法提供正式保证，要么依赖于手工编码的符号执行——这种方法无法扩展到实际策略规范的广度。我们提出了一种自动形式化流水线，该流水线使用基于大语言模型的生成器-批评者循环，将智能体提示、MCP工具描述和自然语言策略文档转换为经过正式验证的策略。生成的策略采用Cedar策略语言编写。在MedAgentBench基准测试中，与先前工作中手工编码的符号执行相比，我们的自动形式化策略覆盖了更多的源自然语言规范。

## Abstract
Agent safety in high-stakes domains requires formal policy enforcement, but most existing approaches either rely on probabilistic guardrails (fine-tuned classifiers, prompt-based steering) that offer no formal guarantees, or on hand-coded symbolic enforcement that does not scale to the breadth of real policy specifications. We present an autoformalization pipeline that translates agent prompts, MCP tool descriptions, and natural language policy documents into formally verified policies using an LLM-based generator-critic loop. The resulting policies are written in the Cedar Policy Language. On the MedAgentBench benchmark, our autoformalized policies cover substantially more of the source natural-language specification than the hand-coded symbolic enforcement in prior work.

### 2. 论文提出的方法论

- **核心思想**：采用“Verification Sandwich”（验证三明治）架构，将形式化策略生成分解为三层：
  - **底层（Grounding Layer）**：从智能体指令和工具模式中提取实体、定义主体-资源-动作本体，并自动生成 Cedar 模式（Schema）。
  - **中间层（Model Layer）**：使用 LLM（Gemini 3 Pro）作为策略生成器，基于输入的系统提示、MCP 工具定义和自然语言策略文档，生成候选的 Cedar 策略集。
  - **上层（Safety Layer）**：采用 **生成器-评论家循环（Generator-Critic Loop）**，包含：
    - **硬批评者（Hard Critic）**：确定性检查，包括 Cedar 语法、模式合规性、矛盾策略和空策略（通过 Cedar 内置静态分析工具）。
    - **软批评者（Soft Critic）**：LLM 作为评判，评估策略与自然语言意图的语义对齐，以及其他定性指标（按评分标准）。
- **关键技术细节**：
  - 目标语言采用 **Cedar 策略语言**，具备可读性、形式验证能力、编译速度快等优点。
  - 循环迭代：若策略未通过任一批评者，则将错误反馈回生成器，重新生成，直到达到质量阈值。
  - 运行时通过外部确定性策略引擎执行，将策略与 LLM 推理上下文解耦，增强抗攻击能力。
