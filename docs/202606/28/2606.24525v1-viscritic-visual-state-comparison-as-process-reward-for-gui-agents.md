## 摘要
由视觉语言模型驱动的GUI智能体在自动化数字任务上展现出巨大潜力，但由于缺乏步骤级验证，在长期任务场景中频繁失败。现有过程奖励模型仅通过文本推理验证动作，忽略了GUI状态变化的视觉本质。我们提出VisCritic，一种视觉过程奖励框架，通过在视觉特征空间中直接比较动作前后的截图来验证智能体动作。VisCritic采用Siamese视觉变换器提取感知变化的表征，并结合动作感知评判头（Action-Aware Critic Head）联合评估动作成功性、任务进度及错误类型。一套评判器训练数据构建流程从现有轨迹中生成弱监督样本，无需额外人工标注即可训练评判器。跨五个基准的实验与离线分析表明，VisCritic可作为多种GUI智能体的即插即用增强模块，在提供视觉诊断线索的同时普遍提升基准指标。

## Abstract
GUI agents powered by vision-language models show strong potential for automating digital tasks, yet frequently fail in long-horizon scenarios due to the absence of step-level verification. Existing process reward models verify actions through textual reasoning alone, missing the visual nature of GUI state changes. We introduce VisCritic, a visual process reward framework that verifies agent actions by directly comparing pre-action and post-action screenshots in visual feature space. VisCritic employs a Siamese vision transformer to extract change-aware representations, coupled with an Action-Aware Critic Head that jointly evaluates action success, task progress, and error type. A critic-training data construction pipeline generates weakly supervised samples from existing trajectories without additional human labels for critic training. Experiments and offline analyses across five benchmarks demonstrate that VisCritic serves as a plug-and-play enhancement for diverse GUI agents, generally improving benchmark metrics while providing visual diagnostic cues.

### VisCritic: Visual State Comparison as Process Reward for GUI Agents 论文分析

#### 1. 核心问题与整体意义（研究动机和背景）

-   **核心问题**：当前由视觉-语言模型驱动的 GUI 智能体在完成复杂的长期任务时，由于缺乏**步骤级验证**机制，经常因为单个错误动作的级联效应而导致任务失败。
-   **现有方法局限**：现有的过程奖励模型（PRMs）主要依赖**文本推理**（如状态描述、工具调用、清单）来验证动作。这与 GUI 交互的**视觉本质**（如按钮颜色变化、弹窗出现、页面滚动）存在根本性的“模态鸿沟”，验证信号不够可靠。
-   **研究目标**：提出一种**以视觉为中心**的验证框架，通过比较动作前后的截图，在视觉特征空间中为 GUI 智能体提供更自然、更有效的过程奖励信号。

#### 2. 方法论：核心思想与技术细节

-   **核心思想**：动作的结果在 GUI 中通常表现为**像素级的状态变化**。通过直接比较动作前后的截图，可以有效判断动作是否产生了预期的视觉效果，从而实现对动作的可靠验证。
-   **关键技术细节**：VisCritic 是一个即插即用的框架，主要由以下三个组件构成：
    1.  **视觉差异编码器（VDE）**：
        -   **孪生特征提取**：使用共享权重的 Vision Transformer 分别对动作前 $s_t$ 和动作后 $s_{t+1}$ 的截图进行编码，得到补丁级特征 $F_t$ 和 $F_{t+1}$。
        -   **差异计算**：计算补丁级差异特征 $\Delta F = F_{t+1} - F_t$ 和变化幅度图 $M$。
        -   **变化区域注意力**：对变化幅度图 $M$ 应用 Softmax 得到注意力权重 $\alpha$，并加权聚合差异特征，得到最终的代表性向量 $v_{\Delta}$。这使得模型能够聚焦于 UI 发生实际变化的区域。
    2.  **动作感知评判头**：
        -   **多任务预测**：融合视觉差异表示 $v_{\Delta}$、动作 $a_t$ 和任务指令 $l$ 的文本特征，进行三个任务的联合预测：
            -   $\hat{y}_{suc}$：动作成功概率（0-1）。
            -   $\hat{y}_{prog}$：任务进度评分（-1 到 1）。
            -   $\hat{y}_{err}$：错误类型分类（如成功、无操作、错点目标等）。
    3.  **弱监督训练数据构建**：
        -   现有轨迹自动生成样本，无需人工额外标注：
            -   **正样本**：来自成功轨迹中的连续截图-动作对。
            -   **负样本**：通过四种策略生成（动作不匹配、状态不匹配、无操作、失败轨迹采样），并自动打上弱标签。

-   **视觉理解瓶颈**：VisCritic 的性能受限于其骨干 VIT 模型的能力，尤其在识别**细微的文本变化**（如计数器从“3”变“4”）或**微妙的 UI 状态转换**方面可能力不从心。
-   **环境敏感性**：任务不相关的视觉动态（如广告动画、异步加载）会干扰验证的准确性。

-   **特定动作类型覆盖**：对于具有可编程验证结果的特定动作类型（如文本输入），视觉比较方法并非最优，与确定性验证器结合会更有效。
