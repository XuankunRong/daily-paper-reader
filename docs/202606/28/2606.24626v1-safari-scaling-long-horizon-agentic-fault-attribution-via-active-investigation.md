## 摘要
随着自主智能体处理日益复杂的多步骤、多智能体任务，其执行轨迹已扩展到超出最大上下文窗口限制的程度。当前有效诊断智能体故障的方法是将完整轨迹加载到LLM的上下文窗口中，但这会遭受注意力稀释，并且当智能体轨迹不可避免地超出上下文限制时，这些方法会失效。为解决这一问题，我们提出了SAFARI（通过主动调查扩展长程智能体故障归因），该框架用一个工具增强的诊断循环替代了线性上下文加载。通过为LLM配备专用工具箱来读取和搜索轨迹片段，并配备持久的短期记忆（STM）以进行跨轮推理，SAFARI有效地将诊断准确性与架构上下文限制解耦。实验表明，在Who&When数据集上，SAFARI在1M token预算内以20%的优势超越了最先进结果；在TRAIL GAIA子集上，在25K token预算内以19%的优势超越最先进结果。最重要的是，即使目标故障位于模型原生上下文窗口5倍之外（传统评估器完全失效的场景），SAFARI仍能保持0.58的精确度。

## Abstract
As autonomous agents tackle increasingly complex multi-step, multi-agent tasks, their execution trajectories have scaled beyond the constraints of even the largest context windows. Current methods for effectively diagnosing agent failures load the full trajectory into an LLM's context window, which suffers from attention dilution and fails when agentic traces inevitably exceed context limits. To address this, we introduce SAFARI (Scaling long-horizon Agentic Fault AttRibution via active Investigation), a framework that replaces linear context loading with a tool-augmented diagnostic loop. By equipping LLMs with a specialized toolbox to read and search trajectory segments alongside a persistent Short-Term Memory (STM) for cross-turn reasoning, SAFARI effectively decouples diagnostic accuracy from architectural context limits. Our experiments demonstrate that SAFARI outperforms state-of-the-art results by 20% on the Who&When dataset within a 1M token budget, and by 19% on TRAIL GAIA subset on a 25K token budget. Most significantly, SAFARI maintains a 0.58 precision even when the target fault resides 5x beyond the model's native context window, a scenario where traditional evaluators fail entirely.

## 2. 论文提出的方法论：核心思想、关键技术细节

### 核心思想
- **主动调查代替线性加载**：不将完整轨迹一次性喂入LLM，而是让**调查智能体（Investigator Agent）**通过迭代工具调用主动搜索、读取轨迹片段，逐步收集证据，最终定位最早且未修正的错误步骤（即“决定性故障”）。
- **短期记忆（Short-Term Memory, STM）**：维护一个持久化的短期记忆块，存储任务目标、假设与证据、调查空白和过去工具调用记录，在每次LLM调用时注入，以克服历史记录因上下文限制被截断造成的信息丢失。

### 关键技术细节
1. **工具设计**：
   - `read(offset, limit)`：从指定偏移量读取轨迹片段（类似于“滚动”日志）。
   - `search(pattern)`：使用不区分大小写的正则表达式搜索轨迹，快速跳转到特定span ID、变量值或错误签名。
2. **推理验证阶段**：调查智能体在得出结论前，必须将假设分解为最多3个原子性声明（atomic claims），并连同证据提交给**推理评估器（Reasoning Evaluator LLM）**。评估器仅根据引用的证据验证每个声明，输出置信度分数（0-100）。若平均置信度≥阈值（默认70），则允许提交结论；否则返回低分声明，智能体需继续搜索证据。
3. **短期记忆（STM）**：包含四个字段——底层任务目标、故障假设与证据、调查空白、过去工具调用。每次LLM调用时附加在历史记录末尾，确保即使早期工具调用被截断，关键信息仍有效。
4. **终止机制**：最多30次迭代（K=30），若超时则执行“强制终结论”合成最佳答案。

### 算法流程
- 初始化：系统提示+用户提示（含轨迹总长度、元数据）。
- 循环：调查智能体基于当前历史H_k和工具模式T产生一个工具调用a，执行后得到观察o，并附加到历史形成H_{k+1}。若历史超出上下文预算，则丢弃最旧轮次。
- STM在每个LLM调用时注入，不存储在H中。
- 当证据足够时，调用evaluate()进行假设验证；若通过则调用submit_step()提交预测。

### 对比方法（Baselines）

- **Step-by-step**：逐步扫描轨迹，每次只处理一个步骤，本质绕过全局上下文限制。
- **RAFFLES**：迭代多轮推理方法（当前SOTA），但限于模型原生上下文窗口。

- **Step-level Accuracy / Tolerant Accuracy**（用于Who&When，允许±s步偏差）。
- **Precision / Strict Precision**（用于TRAIL：预测步骤是否与任意故障对齐 / 是否与高影响故障对齐）。

  - 固定扩展因子（fixed scaling factor）：如ς=1.25, 1.6, 2.5, 5，通过限制模型上下文长度使得故障位置超过预算。
