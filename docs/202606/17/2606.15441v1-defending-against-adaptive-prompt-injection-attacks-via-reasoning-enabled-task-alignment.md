## 摘要
间接提示注入攻击通过将恶意指令嵌入到代理在执行任务期间检索的第三方数据中，劫持基于LLM的代理。现有的防御措施在静态基准测试中报告了接近零的攻击成功率，但最近的自适应评估表明，一旦允许攻击者针对部署的防御进行优化，这些结果就会崩溃。在这项工作中，我们将这种崩溃归因于两种失效模式。首先，现有的防御方法仅限于识别特定的攻击模式，而不是评估每个嵌入指令的意图是否与用户任务相关。其次，基于训练的防御方法（否则会提供最强的安全-效用权衡）从少数手工制作的模板中组装对抗性示例，导致防御者无法在该狭窄的策略分布之外泛化。为了解决这些问题，我们提出了RETA，一种基于训练的方法，将防御决策基于用户任务而非攻击者控制的数据。在每个工具输出步骤中，防御者进行思维链推理，验证其行为与用户任务一致。利用红队，模拟攻击者合成对抗性训练数据并接收字典学习多样性奖励，实现注入重构策略的广泛覆盖。这些共同使得防御者可以通过多目标强化学习进行优化，并获得更好的安全-效用权衡。在六次黑盒自适应攻击中，RETA将每次攻击的ASR保持在10%以下，在两个目标模型上的平均ASR分别为2.92%和3.75%，同时在攻击下和干净输入上保持大部分效用。

## Abstract
Indirect prompt injection attacks hijack LLM-based agents by embedding malicious instructions in third-party data that the agent retrieves during task execution. Existing defenses report near-zero attack success rate on static benchmarks, yet recent adaptive evaluations show that these results collapse once the attacker is allowed to optimize against the deployed defense. In this work, we trace this collapse to two failure modes. First, existing defense methods are confined to recognizing specific attack patterns, rather than assessing whether the intent of every embedded instruction is relevant to the user task. Second, training-based defenses, which otherwise offer the strongest safety-utility trade-off, assemble their adversarial examples from a handful of hand-crafted templates, and the resulting defender fails to generalize outside that narrow strategy distribution. To address these gaps, we propose RETA, a training-based method that grounds defense decisions on the user tasks rather than attacker-controlled data. At each tool-output step, the defender undertakes chain-of-thought reasoning verifying that its actions are consistent with the user task. Leveraging red-teaming, a simulated attacker synthesizes adversarial training data and receives a dictionary-learning diversity reward, achieving broad coverage of injection-reformulation strategies. Together, these allow the defender to be optimized via multi-objective reinforcement learning and achieve better safety-utility trade-off. Across six black-box adaptive attacks, RETA keeps every per-attack ASR below 10%, with average ASR of 2.92% and 3.75% on the two target models, while preserving most utility under attack and on clean inputs.

## 1. 核心问题与整体含义（研究动机与背景）
- **研究动机**：大语言模型（LLM）代理在执行任务时检索外部数据，间接提示注入攻击通过在这些数据中嵌入恶意指令劫持代理。现有防御在静态基准测试中表现优异（ASR接近0%），但在允许攻击者针对防御进行优化的**自适应攻击**下迅速崩溃。
- **关键问题**：防御失效的两大根本原因：
  - **失效模式一**：现有防御局限于识别特定攻击模式（如关键词、结构特征），而非判断指令意图是否与用户任务相关，攻击者通过语义等价改写即可绕过。
  - **失效模式二**：训练类防御的对抗样本仅来自少量手工模板或自适应攻击生成的窄分布，无法覆盖多样化的注入重构策略。
- **整体含义**：提示注入防御必须从“模式识别”转向**任务对齐**——保证代理行为始终服务于可信的用户任务，而非攻击者控制的数据。这要求防御在降低攻击成功率（ASR）的同时，保持攻击下效用（UA）和干净输入效用（BU）。

## 2. 方法论：核心思想、关键技术细节与流程
### 2.1 核心思想
- **原则Ⅰ**：基于用户任务（$t_u$）而非攻击者数据做决策。在每个工具输出步骤，代理通过**思维链推理**验证其行动是否与$t_u$一致。
- **原则Ⅱ**：对抗训练需要策略覆盖（strategy coverage）而非实例覆盖。使用**字典学习多样性奖励**鼓励红队生成未被当前字典良好解释的注入策略。
- **原则Ⅲ**：防御是轨迹级问题，采用**对抗强化学习（Adversarial RL）** 联合优化安全性、效用性和推理质量。

### 2.2 关键技术细节
#### 阶段Ⅰ：红队对抗样本生成
- **红队策略**：用一个基于强化学习的攻击者$π_A$（初始通过SFT热身）生成注入字符串$d_{adv}$，插入到数据通道后，由冻结的基线模型$π_B$完成轨迹。若注入任务成功（$\Phi_{t_{adv}}=1$），则添加到红队集$D_{RT}$。
- **字典学习多样性奖励**：
  - 维护一个可滚动的策略字典$D_ℓ=(V_ℓ, Γ_ℓ)$，其中$V_ℓ$是向量基元，$Γ_ℓ$是文本描述。
  - 对每个新注入，提取策略记录$\psi(d_{adv})$，计算其嵌入$x$，用BPDN求解稀疏编码，得到可解释性得分ES_raw（1~5，LLM评判）。
  - 多样性奖励$r_{div}=1 - \text{ES}$，鼓励生成不被当前词典充分解释的注入。
  - 词典每50步刷新一次，加入原始成功注入和ES<0.5的新成功策略记录。
- **攻击者RL**：使用GDPO优化，奖励包括攻击成功（$r_{atk}$）、多样性（$r_{div}$）和格式（$r_{CoT}$）。

#### 阶段Ⅱ：防御者对抗强化学习
- **防御者初始化**：从边界轨迹（攻击成功且用户任务成功）中提取，由基线模型重写为安全演示（包含任务对齐推理），SFT得到初始防御者$π_D^0$。
- **防御者RL**：从$D_{RT}$采样攻击环境，优化GDPO，奖励包括：
  - 安全奖励$r_{sec}=1[\Phi_{t_{adv}}=0]$（注入任务失败）
  - 效用奖励$r_{utl}=1[\Phi_{t_u}=1]$（用户任务成功）
  - 推理奖励$r_{CoT}$：轨迹中各决策步的推理质量（两个条件C1/C2）的平均LLM评判得分。
- **GDPO**：对每个奖励独立进行组归一化后再加权，防止不同量纲和稀疏性干扰。

### 2.3 算法流程（文字说明）
1. **热身**：攻击者SFT于静态模板；防御者SFT于重写的安全演示。
2. **阶段Ⅰ循环**（每50步一轮）：
   - 学习字典$D_ℓ$
   - 攻击者生成$G_A$组注入，执行基线得到轨迹，收集成功注入（$D_{RT}$），计算多样性奖励，更新攻击者$π_A$
   - 刷新词典：将ES<0.5的成功注入策略加入档案。
3. **阶段Ⅱ循环**（250步）：
   - 从$D_{RT}$采样，防御者$π_D$执行轨迹，计算安全、效用、推理奖励，GDPO更新$π_D$。

## 6. 主要结论与发现
- **RETA在自适应攻击下达到平均ASR 2.92%（Qwen3-4B）和3.75%（Llama-3.1-8B），每个攻击均低于10%**，显著优于最强基线MetaSecAlign（12.5%和5.00%）。
- 同时保持了较高的攻击下效用（UA 55.8%和49.2%）和干净输入效用（BU 60.8%和45.4%），近乎无防御水平。
- **任务对齐推理和多样性奖励是关键**：消融表明，去掉任意一个组件都会使自适应ASR大幅上升（如无推理的SFT + 红队达17.5%，无红队的静态SFT达50%）。
- **跨基准泛化**：RETA在ASB和InjecAgent上静态ASR分别降至9.10%和0.10%，证明其学习到的任务对齐原则可迁移。
- 攻击者需要更多迭代才能成功（中位数19 vs MetaSecAlign的11），说明策略多样性增加了搜索难度。

## 7. 优点
- **方法创新性**：首次将任务对齐推理（Chain-of-Thought）引入提示注入防御，从根本上规避了表面模式绕过的缺陷。
- **策略覆盖机制**：字典学习驱动的多样性奖励，避免了对抗训练中的模式崩塌，使防守泛化更广。
- **多目标RL框架**：GDPO独立归一化各项奖励，直接优化安全-效用-推理三元组，缓解了单目标优化引起的效用下降。
- **实验严谨**：同时覆盖静态和自适应评估，包含大量消融与跨基准测试，对失效模式有深刻分析与可视化（如固定引用可解释性下降对应ASR下降）。
- **代码开源**：提供匿名工件链接，便于复现。

## 8. 不足与局限
- **评估范围有限**：仅考虑单次会话中的一次性注入，未涉及多轮、持久记忆、跨会话或真实动态数据流。
- **自适应攻击子集小**：因计算成本，仅使用40例（而非所有420例静态用例），可能漏掉某些防御脆弱场景。
- **未对比系统级防御**：如CaMeL和IPIGuard的设计范式不同，缺乏公平基准进行比较。
- **推理开销**：任务对齐推理增加了延迟（见表10：Qwen3-4B下Travel套件BU 45% vs 无防御65%，且token和时间翻倍），不适合延迟敏感场景。
- **剩余失效模式**：仍存在两种失效——推理-行动不匹配（模型推理正确但仍执行攻击）和过度防御（将无害参数当作注入拒绝，导致用户任务失败）。作者建议未来引入更细粒度的步骤级奖励。
- **效用边际有时小于某些基线**：如MetaSecAlign的BU略高（64.9% vs 60.8%），说明RETA在安全优先时可能轻微牺牲干净输入效用。
