## 摘要
机器人足球是多智能体强化学习的一个具有挑战性的试验台，因为它结合了部分可观测性、协作与对抗交互、稀疏奖励以及长期战术行为。RoboCup 2D足球仿真（RCSS2D）提供了一个成熟的机器人足球平台，但其面向竞争的服务器-客户端架构难以直接与基于Python的现代MARL工作流配合使用。我们推出了R2D-RL，这是一个强化学习环境，通过共享内存通信和周期级同步，将RCSS2D和基于HELIOS的玩家客户端连接到Python MARL接口。R2D-RL支持全场和基于场景的训练，具有可配置的对手、基础离散和混合参数化动作空间、动作掩码、基于预期控球值（EPV）的奖励塑造以及并行执行。我们提供了前场得分场景和11对11全场基准测试，以及基线结果。

## Abstract
Robot soccer is a challenging testbed for multi-agent reinforcement learning because it combines partial observability, cooperative and adversarial interaction, sparse rewards, and long-horizon tactical behavior. RoboCup 2D Soccer Simulation (RCSS2D) provides a mature robot-soccer platform, but its competition-oriented server-client architecture is difficult to use directly with modern Python-based MARL workflows. We introduce R2D-RL, a reinforcement learning environment that connects RCSS2D and HELIOS-based player clients to a Python MARL interface through shared-memory communication and cycle-level synchronization. R2D-RL supports full-field and scenario-based training with configurable opponents, Base discrete and Hybrid parameterized action spaces, action masks, expected possession value (EPV)-based reward shaping, and parallel execution. We provide front-goal scenarios and an 11-vs-11 full-field benchmark, together with baseline results.

## 1. 核心问题与整体含义（研究动机和背景）

- **动机**：机器人足球是**多智能体强化学习（MARL）** 的理想测试床，具备部分可观测性、协作与对抗交互、稀疏奖励和长期战术行为等挑战。RoboCup 2D足球仿真（RCSS2D）提供了成熟的服务器-客户端架构和HELIOS行为库，但其**面向竞赛的工作流**难以与**现代基于Python的MARL框架**（需步同步、奖励、状态/观察接口、重置控制等）直接集成。
- **背景**：现有相关工作如Google Research Football、NFootball等虽提供MARL友好的接口，但脱离了机器人足球的物理仿真和竞赛规则；Keepaway和Half Field Offense则仅覆盖局部子任务。R2D-RL的目标是**在保留原始RCSS2D完整比赛逻辑和HELIOS行为模块的基础上，构建一个Python MARL环境**，实现全场与场景式学习。

## 2. 方法论

**核心思想**：通过**共享内存通信**和**周期级同步**，将RCSS2D服务器、修改后的HELIOS客户端（玩家、教练、训练员）与Python MARL环境连接，将竞赛工作流转化为**步同步强化学习交互**。

**关键技术细节**：

- **步同步通信协议**：基于序列计数器（`obs_seq`、`act_seq`、`done_seq`）实现。每个环境步对应一个`play_on`模拟周期。玩家发布观察与动作掩码后，环境写入动作并等待确认，确保动作与状态转移一致。
- **RL接口**：
  - **状态**：教练提供136维向量（球位置/速度 + 22个玩家各自6维特征）。
  - **观察**：每个玩家97维（自身、球、11个对手、10个队友特征及游戏模式标识）。
  - **动作空间**：
    - **Base离散空间**：19个高层语义动作（射门、传球、运球、移动等），基于HELIOS行为模块。
    - **Hybrid参数化空间**：5个低级命令（turn, dash, kick, catch）加连续参数。
  - **动作掩码**：基于HELIOS可执行性检查，动态掩蔽不可用动作（如射门优先级掩蔽其他动作）。
  - **奖励**：团队得分奖励（+1/-1） + 可选**MaxEPV塑造奖励**（基于预期控球值的最大增量，只计入控球方）。
  - **终止条件**：进球、出场、失球（对方控球）、超时（步数）。
  - **场景重置**：通过训练员直接移动球和玩家至指定位置，无需重启服务器。
- **并行执行**：支持多个环境实例同时采样，通过共享内存和同步机制实现。

## 6. 主要结论与发现

- **环境可行性**：R2D-RL支持步同步交互、并行采样（Base空间达34.85M步/天，Hybrid达41.65M步/天），并能稳定运行前场和全场任务。
- **组件有效性**：
  - **动作掩码**：在QMIX的Blocked Shot和多个多攻击者场景中至关重要，无掩码时策略常失败。
  - **MaxEPV塑造奖励**：在Complex Defense等高难度场景中显著提升目标率；无EPV时多数设置目标率为零。
- **算法表现**：
  - **MAPPO最稳定**：在多攻击者前场和全场中均领先。
  - **QMIX依赖掩码和EPV**：在简单任务尚可，但在压力下种子方差大。
  - **ParaDQN困难**：Hybrid动作空间在单攻击者简单任务可习得，但在多攻击者几乎为零，表明参数化动作在复杂任务中训练难度大。
- **全场差距**：所有算法目标差均为负，MAPPO最好（-10.61），但仍远弱于HELIOS AI。说明长视界协作需更大规模训练和更先进方法。

## 7. 优点

- **保留原生RCSS2D生态**：直接复用成熟仿真引擎和行为模块，避免从头构建物理引擎。
- **步同步MARL接口**：通过共享内存协议实现可靠的CTDE训练流程，支持重置、动作掩码、奖励塑造等标准RL接口。
- **灵活的动作空间设计**：同时提供高层语义（Base）和低层命令（Hybrid），适应不同抽象层次的研究需求。
- **渐进式场景基准**：从1v0到3v3再到全场，有助于诊断算法在不同复杂度下的能力。
- **组件模块化**：动作掩码和EPV奖励作为可选组件，可清晰分析各自贡献。
- **开源与可复现**：提供代码和基准设置（附录超参数详细）。

## 8. 不足与局限

- **对手固定**：仅使用HELIOS Fallback，对手行为多样性不足，导致过拟合风险。未包含自对弈或对手池。
